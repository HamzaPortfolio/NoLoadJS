Simple SPA Loader Documentation

This is a lightweight JavaScript solution to load multiple HTML pages inside a single index.html without reloading the page. It uses the history.pushState API to update the URL and fetches content dynamically from the pages/ folder.

Project Structure (Required)

For this SPA loader to work correctly, the following project structure is compulsory:

/project
  ├── index.html       <!-- This is your main HTML file where the SPA loader is included -->
  ├── pages/           <!-- Folder containing separate HTML content files -->
  │    ├── about.html
  │    └── services.html

How It Works

  • index.html contains the basic layout and the SPA loader script included via CDN.
  • Links with the class spa-link trigger JavaScript to load corresponding page content inside a content container without full page reload.
  • The URL updates dynamically using history.pushState so users can bookmark or navigate back and forth normally.
  • If the path is /, default content is loaded directly inside index.html.
  • Other paths fetch HTML files from the pages/ folder and inject them inside the content container.

Include the SPA Loader Script (CDN)

Add this line before the closing </body> tag in your index.html to include the loader script:

<script src="https://noloadjs.netlify.app/noload.js"></script>

Example HTML Snippet

<!-- index.html -->
<body>
  <nav>
    <a href="/" class="spa-link">Main Page</a>
    <a href="/about" class="spa-link">About</a>
    <a href="/services" class="spa-link">Services</a>
  </nav>

  <div id="content">
    <!-- Dynamic content will load here -->
  </div>

  <script src="https://noloadjs.netlify.app/noload.js"></script>
</body>

Notes

  • The index.html file must exist in your root folder.
  • The pages/ folder must contain all other HTML content files to be loaded dynamically.
  • Links that you want to load dynamically should have the class spa-link and use relative paths starting with /.
